# 国际化文档

本文档说明前端项目的多语言结构、命名约定和新增文案流程。

## 基本结构

i18n 相关代码位于 `src/i18n`：

- `src/i18n/index.ts`：创建 `vue-i18n` 实例，注册全局语言包。
- `src/i18n/locales/*.ts`：各语言的全局文案。

项目在 [src/main.ts](../src/main.ts) 中注册 i18n。

当前全局语言包注册了 `dev`、`zh-cn`、`de`、`en`、`pl`。`dev` 不包含正式文案，仅用于定位缺失 key。`fr.ts` 文件存在，但当前没有在 `src/i18n/index.ts` 中注册到全局 messages。

## fallback 规则

fallback 配置位于 [src/i18n/index.ts](../src/i18n/index.ts)：

新增任何文案时，至少提供 `zh-cn` 或 `en`，方便后续补译。

## 全局文案

可复用、跨页面或跨组件的文案放在`src/i18n/locales`中，例如菜单、通用按钮、通用错误消息、用户资料字段等。

## 组件局部文案

只在单个组件内使用、复用价值不高的文案，可以直接放在组件内的局部 messages 中。项目惯例是把局部 key 放在 `local` 下。已有示例可参考 [src/components/ExperimentalFeature.vue](../src/components/ExperimentalFeature.vue)。

局部文案建议至少提供 `zh-cn` 或 `en`，方便后续补译。

## 放置规则

- 多处复用的文案放全局语言包。
- 只在一个组件出现的文案放组件局部 messages。
- 表单字段、状态名、通用错误消息优先放全局语言包。
- Cypress测试中无需考虑多语言，但是需要注意：组件测试会渲染英文，而端到端测试会渲染中文。
- 组件局部 key 使用 `local.*`，避免和全局 key 混淆。

## 新增语言

如果不确定应如何新增语言，请直接联系维护团队。

## 翻译检查清单

提交涉及文案的改动前，建议确认：

1. 至少有 `zh-cn` 或 `en` 文案。
2. 局部文案使用 `useI18n({ messages: i18nMessages })`。
3. 文案 key 命名表达业务含义，而不是表达当前显示样式。
4. 没有把可复用文案散落在多个组件局部 messages 中。
