# 国际化文档

本文档说明前端项目的多语言结构、命名约定和新增文案流程。

## 基本结构

i18n 相关代码位于 `src/i18n`：

- `src/i18n/index.ts`：创建 `vue-i18n` 实例，注册全局语言包。
- `src/i18n/config.ts`：定义全局语言包类型 `LocaleConfig`。
- `src/i18n/locales/*.ts`：各语言的全局文案。

项目在 [src/main.ts](../src/main.ts) 中注册 i18n：

```ts
app.use(pinia).use(router).use(i18n);
```

当前全局语言包注册了 `dev`、`zh-cn`、`de`、`en`、`pl`。`fr.ts` 文件存在，但当前没有在 `src/i18n/index.ts` 中注册到全局 messages。

## fallback 规则

fallback 配置位于 [src/i18n/index.ts](../src/i18n/index.ts)：

```ts
fallbackLocale: {
    'zh-cn': ['dev'],
    'en': ['dev'],
    'de': ['en'],
    'pl': ['en'],
    'zh-tw': ['zh-cn'],
}
```

新增全局 key 时，至少补充 `en` 或 `zh-cn`，这样缺失翻译时更容易定位。

## 全局文案

可复用、跨页面或跨组件的文案放在全局语言包中，例如菜单、通用按钮、通用错误消息、用户资料字段等。

示例：

```ts
// src/i18n/locales/en.ts
export default {
    local: 'en',
    common: {
        hide: 'Hide',
    },
};
```

组件中使用：

```ts
import { useI18n } from 'vue-i18n';

const { t } = useI18n();
```

```vue
<template>
    <button>{{ t('common.hide') }}</button>
</template>
```

如果新增全局 key，需要同步更新 [src/i18n/config.ts](../src/i18n/config.ts) 的 `LocaleConfig` 类型，避免语言包结构漂移。

## 组件局部文案

只在单个组件内使用、复用价值不高的文案，可以直接放在组件内的局部 messages 中。项目惯例是把局部 key 放在 `local` 下。

示例：

```ts
import { useI18n } from 'vue-i18n';

const i18nMessages = {
    'zh-cn': { local: {
        watermark: '实验功能',
    } },
    'en': { local: {
        watermark: 'EXPERIMENTAL',
    } },
};

const { t } = useI18n({ messages: i18nMessages });
```

```vue
<template>
    <span>{{ t('local.watermark') }}</span>
</template>
```

局部文案建议至少提供 `zh-cn` 或 `en`，方便后续补译。已有示例可参考 [src/components/ExperimentalFeature.vue](../src/components/ExperimentalFeature.vue)。

## 放置规则

- 多处复用的文案放全局语言包。
- 只在一个组件出现的文案放组件局部 messages。
- 表单字段、状态名、通用错误消息优先放全局语言包。
- 测试中需要稳定查找的文本，优先使用真实用户可见文案，不为测试新增隐藏文案。
- 组件局部 key 使用 `local.*`，避免和全局 key 混淆。

## 新增语言

新增语言时按以下步骤处理：

1. 在 `src/i18n/locales` 下新增语言文件，例如 `fr.ts`。
2. 确保导出的对象包含唯一的 `local` 字段。
3. 在 `src/i18n/index.ts` 中导入并加入 `getMessages()`。
4. 根据需要补充 `fallbackLocale`。
5. 在语言选择 UI 中确认新语言可见。

新增语言后，建议检查页面主流程和设置页，确认切换语言后菜单、弹窗、表单和提示消息都能正常显示。

## 翻译检查清单

提交涉及文案的改动前，建议确认：

1. 新增全局 key 已同步到 `LocaleConfig`。
2. 至少有 `zh-cn` 或 `en` 文案。
3. 局部文案使用 `useI18n({ messages: i18nMessages })`。
4. 文案 key 命名表达业务含义，而不是表达当前显示样式。
5. 没有把可复用文案散落在多个组件局部 messages 中。
